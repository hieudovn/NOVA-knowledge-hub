# Mode Model

## Definition

A Mode is an organizational reasoning perspective.

A Mode is NOT an agent.

A Mode defines how the system should evaluate a situation from a particular business role.

---

## Standard Mode Structure

- Name
- Purpose
- Objectives
- KPIs
- Responsibilities
- Decision Authority
- Constraints
- Communication Style
- Preferred Skills
- Related Artifacts

---

## Initial Modes

### Sales Mode

Focus:
- customer engagement
- qualification
- positioning
- opportunity progression

### Presales Mode

Focus:
- requirement understanding
- solution fit
- compliance
- proposal support

### Technical Mode

Focus:
- architecture
- integration
- implementation feasibility
- risk review

---

## Future Modes

- Product Strategy Mode
- Marketing Mode
- Project Manager Mode
- Executive / CEO Mode

---

## Design Rules

1. Create a Mode only if it represents a real organizational role.
2. Avoid Mode proliferation.
3. Modes should reuse Skills rather than duplicate logic.
4. Modes should be composable by future orchestration.
