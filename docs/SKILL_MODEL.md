# Skill Model

## Definition

A Skill is a reusable business capability.

A Skill converts knowledge into execution.

Skills are reusable across:

- customers
- industries
- projects
- opportunities
- products

---

## Standard Skill Structure

- Name
- Purpose
- Inputs
- Workflow
- Rules
- Output Format
- Quality Criteria
- Related Modes
- Produced Artifact

---

## Principle

Every Skill must produce a concrete artifact.

Examples:

Tender Analyzer
→ Requirement Register

Compliance Matrix Builder
→ Compliance Matrix

Architecture Reviewer
→ Architecture Review Report

Proposal Writer
→ Proposal Draft

---

## Design Rules

1. Skills should be independent of specific customers.
2. Skills should be reusable.
3. Skills should be testable.
4. Skills should produce artifacts.
5. Skills should be callable by future orchestration.
